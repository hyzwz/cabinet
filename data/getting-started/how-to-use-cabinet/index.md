# How to Use Cabinet

Cabinet is an AI-first knowledge base where everything lives as files on disk. There's no database, no cloud lock-in, and no mystery about where your data goes. You write pages in markdown, organize them in a tree, and let AI agents help you edit and maintain the whole thing.

This workspace ships with a working example — the [[Example: Cabinet Carousel Factory]] — a real content pipeline that uses agents, embedded apps, CSVs, PDFs, and linked repos. Every feature described below is demonstrated somewhere in that example, so feel free to poke around as you learn.

## What You'll Find Here

### [[Editor Tips]]
The WYSIWYG editor is where you'll spend most of your time. Rich text formatting, slash commands, markdown toggle, tables, image handling, and auto-save.

### [[AI Editor Guide]]
Claude lives in the right-side panel and can edit your pages directly — no copy-paste required. Learn how to use @mentions to give it context, and craft prompts that get results.

### [[Search and Navigation]]
Cmd+K full-text search across all pages, wiki-links between pages, sidebar drag-and-drop, and more.

### [[Version History]]
Every edit is saved to Git automatically. Browse diffs, see exactly what changed, and restore any previous version with one click.

### [[Apps and Repos]]
Cabinet can embed full web applications, link to external Git repositories, and render interactive tools right in the sidebar. The carousel factory example includes a content calendar, posts editor, and brand kit — all live embedded apps.

### [[Tasks and Jobs]]
A built-in Kanban board for tracking work, plus scheduled jobs that run on cron.

### [[Setup and Deployment]]
How Cabinet initializes your data, where files live on disk, and how to configure a custom data directory.
